# Contributing

We welcome contributions to the Hathor Nano Contracts IDE!

## Development Workflow

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally.
3. **Create a branch** for your feature or fix (`git checkout -b feature/amazing-feature`).
4. **Make your changes**.
5. **Commit your changes** (`git commit -m 'Add some amazing feature'`).
6. **Push to the branch** (`git push origin feature/amazing-feature`).
7. **Open a Pull Request**.

## Coding Standards

- **TypeScript**: We use TypeScript for the frontend. Ensure all types are properly defined.
- **Linting**: Run `npm run lint` to check for linting errors.
- **Formatting**: Code should be formatted according to the project's Prettier configuration.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on the GitHub repository. Provide as much detail as possible, including steps to reproduce the issue.
